# مواصفات نظام ERP للمحلات التجارية

## 1. نظرة عامة

- **اسم النظام**: نظام إدارة المتجر الذكي
- **نوع التطبيق**: تطبيق صفحة واحدة (SPA)
- **الواجهة**: عربية كاملة (RTL)
- **العملة**: الريال اليمني (YER)
- **نمط التخزين**: Offline-First (IndexedDB)

## 2. المواصفات التقنية

### 2.1 تقنيات التخزين
- IndexedDB للبيانات الرئيسية (منتجات، فواتير، عملاء)
- localStorage للإعدادات والبيانات المؤقتة

### 2.2 هيكل البيانات
```
المنتجات:
  - id
  - barcode
  - name
  - category
  - cost_price (بسعر الحبة)
  - retail_price
  - wholesale_price
  - box_price
  - stock (بالحبات)
  - min_stock
  - unit_conversion (1 كرتون = 50 عروسة = 10 حبة لكل عروسة)
  - image
  - created_at
  - updated_at

الفواتير:
  - id
  - invoice_number
  - type (sale/return)
  - items []
  - subtotal
  - discount
  - total
  - payment_method (cash/credit/mixed)
  - customer_name
  - status (completed/cancelled)
  - created_by
  - created_at

المصروفات:
  - id
  - amount
  - category
  - description
  - created_by
  - created_at

العملاء:
  - id
  - name
  - phone
  - balance
  - credit_limit
  - created_at
```

## 3. واجهة المستخدم

### 3.1 الشاشات الرئيسية
1. شاشة الدخول
2. لوحة التحكم
3. نقطة البيع (POS)
4. إدارة المخزون
5. إدارة العملاء
6. التقارير
7. الإعدادات

### 3.2 التصميم
- RTL بالكامل
- ألوان: أزرق رئيسي (#2563eb), أخضر نجاح (#16a34a)
- خلفية رمادية فاتحة (#f8fafc)
- الوضع الداكن مدعوم

## 4. وحدات المنتجات

- **كرتون**: أكبر وحدة
- **عروسة**: وحدة متوسطة (50 حبة = 1 كرتون)
- **حبة**: أصغر وحدة

التحويل:
- 1 كرتون = 50 عروسة
- 1 عروسة = 10 حبات

## 5. الوظائف الرئيسية

### 5.1 نقطة البيع
- بحث بالاسم أو الباركود
- إضافة سريعة للسلة
- وضع البيع/المرتجع
- طرق الدفع: نقدي، آجل، مختلط
- خصومات
- تعليق الفاتورة

### 5.2 المخزون
- تتبع تلقائي للكمية
- تنبيه نقص المخزون
- تحويل الوحدات

### 5.3 المصروفات
- تصنيفات متعددة
- خصم فوري من الصندوق

## 6. التقارير

- تقرير يومي
- تقرير المبيعات
- تقرير الأرباح
- أكثر المنتجات مبيعًا

## 7. البيانات الأولية

منتجات سجائر نموذجية:
- كمران، لوكي، شملان
- أسعار بالريال اليمني
